## 1. 디자인 목표

### 핵심 원칙
- **공장/제조 현장에서 사용하기 쉬운 업무용 대시보드**
- 화려함보다 **가독성, 안정감, 빠른 인지**
- 관리자/현장 직원 모두 이해 가능한 직관적인 UI
- **모바일에서도 사용 가능한 반응형 디자인**

### 사용자 경험 목표
- 데이터 입력은 **3회 클릭 이내**로 완료
- 중요 정보는 **한 눈에 파악** 가능
- NG 비율 등 **위험 지표는 명확하게 강조**
- **에러 메시지가 명확하고 해결 방법 제시**

---

## 2. 타겟 사용자

### 사용자 역할별 특징

#### 현장 작업자 (USER)
- **주요 작업**: 데이터 입력 위주 (OK / NG 기록)
- **필요한 정보**: 오늘의 작업 목록, 입력 화면
- **중요도**: 입력 속도 > 분석 기능

#### 관리자 (MANAGER)
- **주요 작업**: 데이터 조회, 통계 분석, 부품 관리
- **필요한 정보**: 통계, 그래프, 추세 분석, 평가 필요 항목
- **중요도**: 전체 현황 파악 > 세부 정보

#### 최고 관리자 (ADMIN)
- **주요 작업**: 모든 기능 사용, 시스템 관리
- **필요한 정보**: 전체 통계, 사용자 관리, 부품 관리
- **중요도**: 시스템 전반 관리

## 3. 전체 레이아웃 구조

### 기본 레이아웃

```
+--------------------------------------------------------------+
| Header                                                       |
| [Q-Track] | [날짜: 2025-01-18] | [사용자: admin ▼] [로그아웃]|
+----------+---------------------------------------------------+
| Sidebar  | Main Content                                      |
|          |                                                   |
| 메뉴     | +--------------------------------------------+   |
| - 대시보드| | 통계 카드                                  |   |
| - 데이터  | | [총 생산] [총 NG] [평균 NG%] [평가 필요] |   |
|   입력   | +--------------------------------------------+   |
| - 품질   |                                                   |
|   기록   | [공정별 NG 비율 그래프]                           |
| - 부품   |                                                   |
|   관리   | [부품별 NG 비율 그래프]                           |
|          |                                                   |
+----------+---------------------------------------------------+
```

### 레이아웃 컴포넌트 구조

#### Header (상단 헤더)
- **좌측**: 로고/프로그램명 ("Q-Track")
- **중앙**: 현재 날짜 표시
- **우측**: 사용자 정보 + 로그아웃 버튼
- **높이**: 64px (h-16)
- **배경색**: `bg-white border-b border-gray-200`
- **고정**: 페이지 스크롤 시에도 상단 고정

#### Sidebar (좌측 사이드바, 선택사항)
- **너비**: 240px (w-60)
- **배경색**: `bg-gray-50 border-r border-gray-200`
- **메뉴 항목**: 대시보드, 데이터 입력, 품질 기록, 부품 관리 (권한별 표시)
- **활성 메뉴**: `bg-slate-50 text-slate-800`
- **모바일**: 햄버거 메뉴로 토글

#### Main Content (중앙 콘텐츠 영역)
- **패딩**: `p-6` (24px)
- **배경색**: `bg-gray-50` (전체), `bg-white` (카드)
- **최대 너비**: `max-w-7xl mx-auto` (중앙 정렬)
- **반응형**: 모바일에서는 패딩 `p-4`

## 4. 색상 가이드 (Tailwind CSS 기준)

### 🎨 색상 3개 원칙 (핵심)
**예쁜 사이트 = 색이 적다**

#### 1️⃣ Base 색상 (배경, 카드)
- **전체 배경**: `bg-gray-50`
- **카드 배경**: `bg-white`
- **테두리**: `border-gray-200`
- **텍스트 기본**: `text-gray-900`
- **텍스트 보조**: `text-gray-600`
- **텍스트 희미**: `text-gray-400`

#### 2️⃣ Accent 색상 (버튼, 링크 - 1개만)
- **버튼**: `bg-slate-800 hover:bg-slate-900`
- **링크**: `text-slate-700 hover:text-slate-900`
- **강조 요소**: `bg-slate-50 text-slate-800`
- **❌ 주의**: Accent 색상은 **슬레이트/그레이 계열 하나만** 사용 (화려한 색상 금지)
- **💡 참고**: 검정/하얀색 계열과 잘 어울리는 차분한 색상 (inblog.ai 스타일)

#### 3️⃣ State 색상 (상태 표현용)

##### OK (합격/정상)
- **텍스트**: `text-green-600`
- **배경**: `bg-green-50`

##### NG (불량/위험)
- **텍스트**: `text-red-600`
- **배경**: `bg-red-50`

##### 경고 (주의)
- **텍스트**: `text-yellow-600`
- **배경**: `bg-yellow-50`

### ❌ 금지 사항
- **화려한 색상** (파랑, 초록, 보라, 주황 등) 동시 사용 금지
- **평가 필요 항목**: 주황색 대신 슬레이트 계열 사용 (`bg-slate-50 border-l-4 border-slate-400`)
- Accent 색상은 슬레이트/그레이 계열 하나만 사용
- State 색상은 상태 표현용으로만 사용 (버튼 색상 X)

## 5. 타이포그래피

### 📝 글자 크기 계층 (핵심)
**폰트 종류 ❌ / 크기 계층 ⭕**

#### 최소 계층 (3단계)
- **제목**: `text-lg font-semibold` (18px) - 섹션 제목, 카드 제목
- **본문**: `text-sm` 또는 `text-base` (14px / 16px) - 기본 텍스트
- **보조 설명**: `text-xs text-gray-500` (12px) - 설명, 부가 정보

#### 추가 계층 (선택)
- **페이지 제목**: `text-2xl font-bold` (24px) - 최상위 제목만
- **숫자/통계 강조**: `text-xl font-bold` (20px) - KPI 숫자

**💡 핵심**: 글자가 크다고 강조되는 게 아니라, 다른 글자보다 클 때 강조됨

### 폰트 패밀리
- 기본: 시스템 폰트 (`sans-serif`)
- 숫자: `tabular-nums` (숫자 정렬)

### 줄 간격 (Line Height)
- **본문**: `leading-normal` (1.5배) - 기본 권장값
- **제목**: `leading-tight` (1.25배) - 제목용
- **💡 권장**: 폰트 크기의 1.4~1.6배가 가독성에 좋음

### 정렬 원칙
- **텍스트**: 무조건 왼쪽 정렬 (`text-left`)
- **카드 묶음**: 그리드 정렬 (`grid`)
- **숫자 KPI**: 중앙 정렬 가능 (`text-center`)
- **❌ 금지**: 어중간한 정렬 (버튼 중앙, 텍스트 좌측, 아이콘 우측 혼재)

## 6. 공통 컴포넌트 스타일

### 📦 카드 (Card) - 기본 단위
**요즘 UI = 카드의 조합**

```jsx
<div className="bg-white rounded-xl shadow-sm p-4 border border-gray-200">
  {/* 카드 내용 */}
</div>
```

#### 카드 공식
- **배경**: `bg-white`
- **라운드**: `rounded-xl` (12px)
- **그림자**: `shadow-sm` (있다/없다만) - 중간 그림자 금지
- **패딩**: `p-4` ~ `p-6` (카드 내부 여백)
- **테두리**: `border border-gray-200` (선택사항)

#### 카드 사용 원칙
- **정보 묶임**: 관련 정보를 카드로 묶기
- **여백 자동 확보**: 카드 사이 간격은 `gap-4` 이상
- **정렬**: 그리드로 자동 정렬

### 📏 여백(Spacing)이 디자인의 50%
**여백 = 고급스러움**

#### 실무 기준
- **카드 내부 padding**: `p-4` ~ `p-6`
- **섹션 간 간격**: `space-y-6` 이상
- **화면 가장자리**: 항상 여유 (`p-6` 또는 `px-4 py-6`)
- **요소 사이 간격**: `gap-4`, `gap-6` 적극 사용

**💡 Tailwind에서 예쁜 사이트 = `p-`, `gap-`, `space-`가 많다**

### 버튼 스타일

#### 기본 버튼 (Accent 색상 사용)
```jsx
<button className="px-4 py-2 bg-slate-800 text-white rounded-lg hover:bg-slate-900 transition-colors">
  저장
</button>
```

#### 버튼 원칙
- **Accent 색상**: 모든 버튼은 `bg-slate-800` 사용 (색상 통일)
- **취소/보조**: `bg-gray-300 hover:bg-gray-400 text-gray-800`
- **❌ 금지**: 버튼마다 다른 색상 사용 (화려한 색상 금지)

#### 버튼 크기
- **작은 버튼**: `px-3 py-1.5 text-sm`
- **기본 버튼**: `px-4 py-2 text-base`
- **큰 버튼**: `px-6 py-3 text-lg`

#### 상태 표현 버튼 (State 색상은 버튼에 사용 안 함)
- **OK/NG는 텍스트나 배경으로만 표현**
- 버튼 색상은 항상 슬레이트/그레이 계열 (Accent)

### 입력 필드 (Input)
```jsx
<input className="w-full px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-slate-500 focus:border-transparent" />
```
- **기본 스타일**: `border border-gray-300 rounded-lg`
- **포커스**: `focus:ring-2 focus:ring-slate-500`
- **에러 상태**: `border-red-500`

### 드롭다운 (Select)
```jsx
<select className="w-full px-4 py-2 border border-gray-300 rounded-lg bg-white focus:outline-none focus:ring-2 focus:ring-slate-500">
  {/* 옵션 */}
</select>
```

### 페이지 구조 및 일관성 규칙

#### 📌 핵심 원칙
**페이지 내부도 "일관성"이 중요하다**

모든 페이지가 동일한 구조와 스타일을 유지해야 사용자가 혼란 없이 사용할 수 있습니다.

---

### 📌 페이지 제목 규칙

#### 위치 및 스타일
- **위치**: 항상 왼쪽 상단 (Main Content 영역 최상단)
- **스타일**: `text-lg font-semibold text-gray-900`
- **규칙**: 페이지당 제목 1개만

```jsx
// 페이지 제목 예시
<h1 className="text-lg font-semibold text-gray-900 mb-6">
  데이터 입력
</h1>
```

#### ❌ 금지
- 페이지 중앙에 제목 배치
- 제목 여러 개 사용
- 제목 크기 제각각 (`text-2xl`, `text-3xl` 등 혼용 금지)

---

### 📌 액션 버튼 위치 규칙

#### 위치 고정
- **저장/확인 버튼**: 항상 **오른쪽 상단** 또는 **오른쪽 하단**
- **취소 버튼**: 저장 버튼 왼쪽에 배치
- **일관성**: 모든 페이지에서 동일한 위치 유지

```jsx
// 버튼 위치 예시 (상단)
<div className="flex justify-end gap-2 mb-6">
  <button className="px-4 py-2 bg-gray-300 text-gray-800 rounded-lg">
    취소
  </button>
  <button className="px-4 py-2 bg-slate-800 text-white rounded-lg">
    저장
  </button>
</div>

// 또는 하단
<div className="flex justify-end gap-2 mt-6">
  <button>취소</button>
  <button>저장</button>
</div>
```

#### ❌ 금지
- 버튼 위치를 페이지마다 다르게 배치
- 왼쪽 상단에 액션 버튼 배치 (제목과 겹침)
- 중앙 정렬된 버튼 (일관성 저해)

---

### 📌 카드 사용 규칙

#### 원칙
- **모든 정보는 카드 안에**: 콘텐츠는 반드시 카드로 감싸기
- **카드 모서리 동일**: 모든 카드는 `rounded-xl` 사용
- **카드 그림자 동일**: 모든 카드는 `shadow-sm` 사용

```jsx
// 올바른 예시
<div className="bg-white rounded-xl shadow-sm p-4 border border-gray-200">
  {/* 콘텐츠 */}
</div>

// ❌ 잘못된 예시
<div className="bg-white rounded-lg shadow-md p-4">
  {/* 카드 스타일이 일관되지 않음 */}
</div>
```

#### ❌ 금지
- 카드 없이 직접 콘텐츠 표시 (헤더, 푸터 제외)
- 페이지마다 카드 스타일 다르게 (`rounded-lg`, `rounded-md` 혼용 금지)
- 그림자 크기 혼용 (`shadow-sm`, `shadow-md` 혼용 금지)

---

### 🚨 예외는 언제 허용되나?

#### 딱 2가지 경우만 예외 허용

##### 1️⃣ 로그인 페이지 (`/login`, `/signup`)
- **Sidebar 없음**: 전체 화면 사용
- **Header 없음**: 최소 UI
- **카드 중앙 배치**: 로그인 폼만 중앙 정렬
- **이유**: 인증 전 페이지이므로 레이아웃 제약 없음

```jsx
// 로그인 페이지 레이아웃 예시
<div className="min-h-screen flex items-center justify-center bg-gray-50">
  <div className="bg-white rounded-xl shadow-sm p-8 w-full max-w-md">
    {/* 로그인 폼 */}
  </div>
</div>
```

##### 2️⃣ 에러 / 알림 전용 페이지 (`/404`, `/500`, `/error`)
- **최소 UI**: 에러 메시지만 표시
- **헤더/사이드바 없음**: 오류 상황 명확화
- **이유**: 예외 상황 처리 페이지

```jsx
// 에러 페이지 레이아웃 예시
<div className="min-h-screen flex items-center justify-center bg-gray-50">
  <div className="text-center">
    <h1 className="text-2xl font-bold text-gray-900 mb-2">404</h1>
    <p className="text-gray-600">페이지를 찾을 수 없습니다.</p>
  </div>
</div>
```

#### 그 외는 ❌ 예외 만들지 마라
- 대시보드, 데이터 입력, 품질 기록, 부품 관리 등 **모든 일반 페이지**는 동일한 레이아웃 유지
- 페이지마다 구조를 다르게 만들지 않기
- **일관성 > 개성**

---

### 🧠 개발 관점에서도 이득이 크다

#### 구조가 같으면
- **Layout 컴포넌트 1개**: 모든 페이지가 공통 Layout 사용
- **페이지는 내용만 작성**: 레이아웃 로직 불필요
- **유지보수 쉬움**: 한 곳만 수정하면 모든 페이지에 적용

- **업계 평균**: `rgba(148, 163, 184, 0.5)` (slate-400, 기준선 - 차분한 그레이)

**💡 참고**: 그래프는 검정/하얀색 계열과 어울리는 차분한 색상 사용 (화려한 색상 지양)

#### 그래프 옵션
- **그리드**: 눈금선 표시 (`display: true`)
- **범례**: 우측 상단 표시
- **툴팁**: 데이터 표시
- **반응형**: `responsive: true`

## 11. UX 원칙

### 정보 구조
- ✅ **한 화면에 핵심 정보만**: 최대 3-5개 주요 정보
- ✅ **계층적 구조**: 중요도에 따른 크기/색상 차별화
- ✅ **일관된 네비게이션**: 항상 같은 위치에 메뉴/버튼

### 상호작용 및 피드백 (Feedback)

#### 즉각적인 피드백
- ✅ **버튼 클릭**: `hover:bg-slate-900 transition-colors` (시각적 상태 변화)
- ✅ **로딩 상태**: 스피너 또는 스켈레톤 UI 표시 (`isLoading` 상태)
- ✅ **성공 메시지**: 녹색 배경 + 확인 아이콘 (`bg-green-50 text-green-600`)
- ✅ **에러 메시지**: 빨간 배경 + 경고 아이콘 (`bg-red-50 text-red-600`)

#### 확인 다이얼로그
- ✅ **삭제/중요 액션**: 확인 다이얼로그 필수
- ✅ **취소 가능**: 모든 모달/다이얼로그는 취소 버튼 제공

#### 입력 유효성 검사
- ✅ **실시간 검증**: 입력 중 즉시 피드백 (에러 메시지)
- ✅ **에러 위치 강조**: 해당 입력 필드에 빨간 테두리 (`border-red-500`)
- ✅ **성공 표시**: 유효한 입력 시 확인 아이콘 (선택)

#### 단축키 지원 (선택사항)
- ✅ **Enter**: 폼 제출 / 저장
- ✅ **Esc**: 모달/다이얼로그 닫기
- ✅ **Tab**: 다음 필드로 이동

### 에러 처리
- ✅ **명확한 에러 메시지**: 무엇이 잘못되었는지 구체적으로
- ✅ **해결 방법 제시**: 어떻게 고칠 수 있는지 안내
- ✅ **에러 위치 강조**: 해당 입력 필드에 빨간 테두리

### 접근성 (Accessibility)

#### 색상 대비 (Contrast Ratio)
- ✅ **WCAG AA 기준 준수**: 텍스트와 배경 간 대비 비율 최소 **4.5:1** 이상
- ✅ **큰 텍스트**: 18px 이상 또는 굵은 글씨는 **3:1** 이상
- ✅ **상태 표시**: 색상만으로 정보를 전달하지 않기 (아이콘, 텍스트 병행)

#### 키보드 네비게이션
- ✅ **Tab 키 접근**: 모든 인터랙티브 요소에 접근 가능
- ✅ **포커스 표시**: 명확한 포커스 링 표시 (`focus:ring-2 focus:ring-slate-500`)
- ✅ **논리적 순서**: Tab 키 순서가 화면 흐름과 일치

#### 스크린 리더 지원
- ✅ **텍스트 대안**: 아이콘에는 `aria-label` 또는 텍스트 레이블 제공
- ✅ **시맨틱 HTML**: 적절한 HTML 태그 사용 (`<button>`, `<nav>`, `<main>` 등)
- ✅ **폼 레이블**: 모든 입력 필드에 `<label>` 태그 연결

**💡 권장 도구**: [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)로 색상 대비 확인

## 14. 아이콘 사용 원칙

### 🎯 아이콘은 "의미 전달용"으로만
**아이콘은 장식이 아니다**

#### 사용 용도
- **버튼 왼쪽에 작은 아이콘**: 액션의 의미 전달
- **상태 표시용 아이콘**: OK/NG/경고 표현
- **탭/메뉴 아이콘**: 네비게이션 보조

#### 권장 아이콘 (Heroicons 등)
- **대시보드**: `HomeIcon`
- **입력**: `PlusCircleIcon`
- **통계**: `ChartBarIcon`
- **설정**: `CogIcon`
- **경고**: `ExclamationTriangleIcon`
- **확인**: `CheckCircleIcon`

#### ❌ 금지 사항
- **의미 없는 아이콘 남발**: 장식용 아이콘 금지
- **아이콘 크기 제각각**: 통일된 크기 사용 (`w-5 h-5` 또는 `w-4 h-4`)
- **아이콘만 사용**: 텍스트 레이블과 함께 사용 권장

- **과도한 애니메이션**: 성능 저하, 산만함
- **자동 움직이는 요소**: 사용자 주의 분산
